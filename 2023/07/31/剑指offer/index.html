153. 寻找旋转排序数组中的最小值

解法：二分法
算法描述：
遇见排序数组查找，就想二分，该题数组内没有重复值，所有只有两种情况，第一种情况，mid>nums[right]说明mid在左升区间内，由于旋转，左升区间内的所有元素大于右区间。所以left = mid+1.第二种情况，mid<nums[right],说明mid在右区间，由于右区间也是递增区间，所以mid右边都比mid的元素大，所以right = mid。当left==right，说明找到最小值。

为什么right=mid，而不是right = mid+1？
因为我们要找的是最小值，mid在右区间内，mid也可能是右区间第一个元素，也就是最小值，所以right = mid；
为什么mid不和left进行比较？
因为我们找的是最小值，偏向左边找，如果和left进行比较，会出现很多区间问题，判断麻烦，
而和right比较，不需要考虑区间问题，思路更清晰。
二分目的是判断 m 在哪个排序数组中，从而缩小区间。而在 nums[m]>nums[i情况下，无法判断 m 在哪个排序数组中。本质上是由于 j 初始值肯定在右排序数组中； i 初始值无法确定在哪个排序数组中。举例如下：

为什么numbers[m]==number[j]时要j–？
numbers[m]==number[j]有两种情况，第一种mid是最小值，则j也是最小值，则j–没有影响。
第二种情况：mid不是最小值，则j不是最小值，即j–即可排除一个错误答案，继续缩小区间，进行二分。

模板：
统一使用 while (left < right) ：

  • 寻找第一个满足XXX的位置：

  • 寻找最后一个满足XXX的位置：

  • 最终的二分结果，返回 left 或 right 都可，因为退出循环时一定有 left==right 。

你可以把这两种模板都记住，也可以只选择其中一套看着更顺眼的。因为这两种模板之间的结果是可以互相转化的。

假设你只记住了 寻找第一个满足XXX 这个模板，但是遇到的实际需求是 寻找最后一个满足XXX ，你完全可以将二分目标修改为： 寻找第一个不满足XXX 的位置（也就是把 if条件 改成对应相反的，其余都不用动）。可知它的上一个位置就是 最后一个满足XXX 的，所以你把算出来的二分结果减去1，就是答案了。（反之同理，由 最后一个 -> 第一个 是 +1 操作）

不过，这样得到的结果，很多时候都需要进行 后判断处理 ，否则会带来问题。

  • 如果你是由 一种情况 转化到 另一种情况 的话，可能会带来 “索引越界” 的问题。这是因为：在结果转化时需要进行 -1(或者 +1) 的操作。假设本身的二分结果是0，然后你减去了1，结果变成-1了，那就下标越界啦！所以在最后返回前，必须判断一下这个下标究竟是否合法（如果题目保证一定存在答案，那么大可放心不必进行这些判断）：

  • 另外，如果题目不保证一定存在答案，那么即使没有产生 “索引越界” ，也可能得到 “错误答案” 。例如 寻找有序数组中元素值为 target 的元素所在位置，但是并不保证它一定存在 。拿到这题，我设定二分目标是： 寻找第一个满足 nums[i] >= target 的位置 i ，最终我得到的二分结果是 left (也同样是 right ) 。可是我只知道 nums[left] >= target 是没错的，但不见得 nums[left] == target 也一定没错！！因此就必须要后判断处理：

剑指 Offer 38. 字符串的排列

经典的回溯算法题
解法：回溯+剪枝
排列方案的生成：
根据字符串排列的特点，考虑深度优先搜索所有排列方案。即通过字符交换，先固定第 1 位字符（ n 种情况）、再固定第 2 位字符（ n−1 种情况）、… 、最后固定第 n 位字符（ 111 种情况）。简单来说就是每层递归固定一位。
重复排列方案与剪枝：
当字符串存在重复字符时，排列方案中也存在重复的排列方案。为排除重复方案，需在固定某位字符时，保证 “每种字符只在此位固定一次” ，即遇到重复字符时不交换，直接跳过。从 DFS 角度看，此操作称为 “剪枝”

剑指 Offer 07. 重建二叉树

解法：分治算法
根据以上性质，可得出以下推论：
前序遍历的首元素 为 树的根节点 node 的值。
在中序遍历中搜索根节点 node 的索引 ，可将 中序遍历 划分为 [ 左子树 | 根节点 | 右子树 ] 。
根据中序遍历中的左（右）子树的节点数量，可将 前序遍历 划分为 [ 根节点 | 左子树 | 右子树 ]
通过以上三步，可确定 三个节点 ：1.树的根节点、2.左子树根节点、3.右子树根节点。
总结：
这道题其实就是寻找三个节点：1.根节点，2.左子树的根节点，3.右子树的根节点
根节点＝当前节点
左子树的根节点＝前序遍历中当前节点+1（这里不理解的话就多顺着树图念几遍根左右根左右…）
右子树的根节点＝当前节点+左子树长度+1
（见k神图1中的3）

其实这也正是前序和中序遍历数组各自提供的功能，前序遍历用来定位根节点，中序遍历用来定位左右子树边界。

剑指 Offer 56 - I. 数组中数字出现的次数

解法：位运算
算法描述：
关键：异或运算两个相同的数字异或为0，并且异或运算满足交换率a⊕b=b⊕a，即以上运算结果与 nums的元素顺序无关。
如果只有一个不重复值，则全员异或得到的是该不重复值；
如果有两个，则得到的值为两个不重复值的异或结果。对异或结果进行与1进行与运算，因为两个值都是不重复值，所以异或结果一定有位数为1，然后找到为1的值，根据此值对nums进行分组，可以将数组种的两个不重复值分隔在两个数组中，然后每个数组就都只有一个不重复值；对这两个数组分别进行全员异或；就得到了nums中的两个不重复值
